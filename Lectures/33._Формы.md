
# React Hook Form

---

# 1️⃣ Зачем вообще нужна библиотека для форм?

В обычном React форма выглядит так:

- useState для каждого input
    
- onChange
    
- хранение ошибок
    
- валидация
    
- обработка submit
    

Когда полей становится 5–10 — код начинает разрастаться.

React Hook Form решает проблемы:

✅ Меньше кода  
✅ Меньше перерисовок  
✅ Встроенная валидация  
✅ Удобная работа с ошибками  
✅ Высокая производительность

---

# 2️⃣ Установка

```bash
npm install react-hook-form
```

---

# 3️⃣ Базовый пример

```jsx
import { useForm } from "react-hook-form";

function MyForm() {
  const { register, handleSubmit } = useForm();

  const onSubmit = (data) => {
    console.log(data);
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("name")} placeholder="Имя" />
      <input {...register("email")} placeholder="Email" />
      <button type="submit">Отправить</button>
    </form>
  );
}
```

---

## Что здесь происходит?

### useForm()

Главный хук библиотеки.  
Он возвращает:

- register — подключает input к форме
    
- handleSubmit — обработка отправки
    
- formState — ошибки и состояние формы
    
- reset — очистка формы
    
- watch — отслеживание значений
    

---

# 4️⃣ Валидация

Добавим проверку:

```jsx
<input
  {...register("email", {
    required: "Email обязателен",
    pattern: {
      value: /^\S+@\S+$/i,
      message: "Неверный формат email"
    }
  })}
/>
```

Добавим вывод ошибок:

```jsx
const { register, handleSubmit, formState: { errors } } = useForm();
```

```jsx
{errors.email && <p>{errors.email.message}</p>}
```

---

## Доступные правила:

- required
    
- minLength
    
- maxLength
    
- pattern
    
- validate (своя функция)
    

---

# 5️⃣ Почему библиотека быстрая?

Обычный React:

→ каждый onChange вызывает setState  
→ компонент перерисовывается

React Hook Form:

→ использует uncontrolled inputs  
→ минимизирует ререндеры  
→ хранит данные через ref

Это делает формы быстрее.

---

# 6️⃣ reset()

Очистка формы:

```jsx
const { reset } = useForm();

reset(); 
```

Можно передать новые значения:

```jsx
reset({
  name: "Alex",
  email: "alex@mail.com"
});
```

---

# 7️⃣ watch()

Следим за значением поля:

```jsx
const { watch } = useForm();

const name = watch("name");
```

Полезно для:

- live-preview
    
- динамических форм
    
- условного рендера
    

---

# 8️⃣ Controller (для сложных компонентов)

Если вы используете:

- кастомные компоненты
    
- UI-библиотеки
    
- datepicker
    

Тогда нужен Controller:

```jsx
import { Controller } from "react-hook-form";

<Controller
  name="date"
  control={control}
  render={({ field }) => (
    <input type="date" {...field} />
  )}
/>
```
