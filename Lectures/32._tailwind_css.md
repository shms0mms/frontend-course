
## 1. Проблема: Почему обычный CSS — это иногда «боль»?

Вспомните, как мы делали раньше:

1. Придумываем название класса (например, `super-mega-button-container`).
    
2. Идем в отдельный файл `.css`.
    
3. Пишем там 5-10 строк кода.
    
4. Возвращаемся в React, понимаем, что забыли отступ, и идем обратно.
    

**Tailwind** — это подход «Utility-first» (утилиты в первую очередь). Вместо того чтобы придумывать названия, мы просто вешаем на элемент готовые «наклейки» (классы), которые делают ровно одну вещь: `bg-blue-500` (синий фон), `p-4` (отступ).

---

## 2. Основные «кубики» Tailwind

Давайте разберем самые частые классы, которые заменят вам сотни строк CSS:

### Размеры и отступы

- **Padding (внутренние):** `p-2`, `px-4` (влево-вправо), `py-8` (вверх-вниз).
    
- **Margin (внешние):** `m-4`, `mt-2` (сверху), `mx-auto` (центрирование).
    
- **Ширина и высота:** `w-full`, `w-1/2`, `h-screen` (на весь экран), `w-[300px]` (произвольный размер).
    

### Цвета и текст

- **Цвет текста:** `text-red-500`, `text-white`.
    
- **Размер шрифта:** `text-sm`, `text-xl`, `text-4xl`.
    
- **Фон:** `bg-green-200`, `bg-indigo-600`.
    

### Flexbox (самое важное!)

Вместо того чтобы писать 3 строки в CSS, пишем прямо в `className`:

- `flex` — включаем флекс.
    
- `items-center` — выравниваем по вертикали.
    
- `justify-between` — расталкиваем элементы по краям.
    
- `gap-4` — расстояние между элементами.

## 3. Интерактив и Состояния

В Tailwind очень легко делать эффекты при наведении. Просто добавьте приставку `hover:` к любому классу.

```js
<button className="bg-blue-500 hover:bg-blue-700 transition-colors p-4 rounded-lg">
  Нажми на меня!
</button>
```

- `transition-colors` — делает смену цвета плавной.
    
- `rounded-lg` — закругленные углы.
