# Интенсивный План Курса по React (2 урока/неделя x 2 часа, с Разбитой Практикой)

Этот план курса разработан с учетом интенсивного темпа, 2 уроков в неделю по 2 часа. Особое внимание уделяется **практике**, которая разбита на более мелкие, управляемые блоки в рамках каждого урока.

---

## Неделя 1: Быстрый Старт с React

*   **Урок 1: Основы React, JSX и Компоненты (2 часа)**
    *   **Теория (40 мин):** Что такое React, SPA, Настройка проекта (Vite/CRA). JSX - синтаксис и использование. Функциональные компоненты и Пропсы.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Создание Проекта и Первого Компонента (25 мин)**
            *   Создание нового React-проекта (через командную строку).
            *   Очистка стандартного кода, создание своего компонента `App`.
            *   Написание простого "Привет, мир!" в JSX.
            *   Запуск сервера разработки и проверка результата.
        *   **Обсуждение и Вопросы (5 мин):** Краткое обсуждение, ответы на вопросы по настройке и первому коду.
        *   **Задача 2: Создание Нескольких Компонентов (25 мин)**
            *   Создание 2-3 новых простых функциональных компонентов (например, `<Header>`, `<Footer>`, `<Greeting>`).
            *   Импорт и использование этих компонентов в компоненте `App`.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение структуры компонентов, ответов на вопросы по импорту/экспорту.
        *   **Задача 3: Введение в Пропсы (25 мин)**
            *   Добавление пропсов к одному из созданных компонентов (например, передача `name` в `<Greeting>`).
            *   Использование переданных пропсов внутри компонента.
            *   Рендеринг компонента с разными значениями пропсов.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение работы пропсов, типовых ошибок.

*   **Урок 2: Состояние и События (2 часа)**
    *   **Теория (40 мин):** Состояние (State) с хуком `useState`. Обновление состояния (прямое и по функции). Обработка событий в React. Синтетические события.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Добавление Состояния (30 мин)**
            *   Создание компонента с состоянием (например, булево для видимости).
            *   Использование состояния для условного рендеринга.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение `useState`.
        *   **Задача 2: Создание Счетчика (35 мин)**
            *   Реализация простого счетчика с использованием состояния и кнопок "+", "-".
            *   Обработка кликов по кнопкам.
            *   Обновление состояния с учетом предыдущего значения.
        *   **Обсуждение и Вопросы (5 мин):** Разбор решений, обсуждение обновления состояния.
        *   **Задача 3: Обработка Ввода (15 мин)**
            *   Создание поля ввода (`<input type="text">`).
            *   Отображение текста из поля ввода по мере его изменения (с помощью состояния и обработчика `onChange`).
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение работы с полями ввода.

## Неделя 2: Условия, Списки и Эффекты (Основы)

*   **Урок 3: Условный Рендеринг и Списки (2 часа)**
    *   **Теория (40 мин):** Условный рендеринг (&&, ?:). Рендеринг списков с `map`. Важность свойства `key`. Ошибки, связанные с `key`.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Условное Отображение (30 мин)**
            *   Показать/скрыть блок по клику на кнопку.
            *   Рендеринг разного контента в зависимости от булева состояния.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение условных операторов в JSX.
        *   **Задача 2: Рендеринг Простого Списка (35 мин)**
            *   Отображение списка строк из массива с использованием `map`.
            *   Правильное использование `key`.
        *   **Обсуждение и Вопросы (5 мин):** Разбор решений, обсуждение `key`.
        *   **Задача 3: Рендеринг Списка Объектов (15 мин)**
            *   Создание массива объектов (например, пользователей с id и именем).
            *   Отображение списка пользователей с использованием `map` и компонента для элемента списка.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение рендеринга более сложных данных.

*   **Урок 4: Хук `useEffect` (Начало) (2 часа)**
    *   **Теория (40 мин):** Понятие побочных эффектов в React. Использование `useEffect` для выполнения действий после рендеринга. Массив зависимостей (пустой - только при монтировании). Очистка эффектов (cleanup function).
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Эффект при Монтировании (35 мин)**
            *   Использование `useEffect` с пустым массивом зависимостей.
            *   Пример: изменение заголовка страницы (`document.title`) при загрузке компонента.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение выполнения эффектов при монтировании.
        *   **Задача 2: Очистка Эффекта (35 мин)**
            *   Реализация эффекта с очисткой (например, установка и очистка интервала или таймера).
            *   Демонстрация работы очистки при размонтировании компонента.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение важности очистки эффектов.

## Неделя 3: Загрузка Данных и Формы

*   **Урок 5: Получение Данных с Сервера (2 часа)**
    *   **Теория (40 мин):** Асинхронные операции в React. Использование `fetch` или Axios в `useEffect` для загрузки данных. Обработка состояния загрузки (`loading`) и ошибок (`error`).
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Базовая Загрузка (40 мин)**
            *   Создание компонента, который загружает данные с тестового API (например, `https://jsonplaceholder.typicode.com/users`) при монтировании с использованием `useEffect` и `fetch`.
            *   Сохранение полученных данных в состоянии.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение асинхронности, `useEffect`.
        *   **Задача 2: Индикатор Загрузки и Ошибки (35 мин)**
            *   Добавление состояния `isLoading` и `error`.
            *   Отображение индикатора загрузки во время выполнения запроса.
            *   Отображение сообщения об ошибке, если запрос не удался.
        *   **Обсуждение и Вопросы (5 мин):** Обработка состояний запроса.

*   **Урок 6: Работа с Формами (2 часа)**
    *   **Теория (40 мин):** Управляемые компоненты форм. Привязка полей ввода к состоянию. Обработка событий `onChange`. Обработка отправки формы `onSubmit`. Предотвращение перезагрузки.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Создание Простой Формы (40 мин)**
            *   Создание компонента формы с несколькими полями ввода (например, имя, email).
            *   Создание состояния для каждого поля ввода.
            *   Связывание значений полей с состоянием через обработчик `onChange`.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение связи полей ввода и состояния.
        *   **Задача 2: Отправка Формы и Базовая Валидация (35 мин)**
            *   Добавление кнопки отправки.
            *   Реализация обработчика `onSubmit`.
            *   Вывод данных из формы при отправке.
            *   Реализация базовой валидации (например, проверка, что поля не пустые) и отображение сообщения об ошибке, если валидация не пройдена.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение отправки форм и валидации.

## Неделя 4: Передача Данных и Маршрутизация

*   **Урок 7: Контекст API и `useContext` (2 часа)**
    *   **Теория (40 мин):** Проблема "Prop Drilling". Введение в Context API. `React.createContext`, `Provider`, `useContext`. Когда использовать контекст.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Создание и Предоставление Контекста (40 мин)**
            *   Создание нового контекста (например, `ThemeContext`).
            *   Создание компонента-провайдера (`ThemeProvider`).
            *   Обертывание части приложения в провайдер и передача значения.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение Context API.
        *   **Задача 2: Использование Контекста с `useContext` (35 мин)**
            *   В компонентах, вложенных в провайдер, использовать хук `useContext` для получения значения контекста.
            *   Изменение UI в зависимости от значения контекста (например, смена темы).
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение `useContext` и примеров использования.

*   **Урок 8: Маршрутизация с React Router (2 часа)**
    *   **Теория (40 мин):** Основы маршрутизации в SPA. Установка `react-router-dom`. Компоненты `BrowserRouter`, `Routes`, `Route`. Навигация с `Link`.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Базовая Настройка и Маршруты (40 мин)**
            *   Установка `react-router-dom`.
            *   Оборачивание приложения в `BrowserRouter`.
            *   Создание нескольких простых компонентов-страниц (например, Home, About, Contact).
            *   Определение маршрутов для этих страниц с использованием `Routes` и `Route`.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение базовой маршрутизации.
        *   **Задача 2: Навигация и Параметры Маршрута (35 мин)**
            *   Создание навигационного меню с использованием компонента `Link`.
            *   Создание маршрута с параметром (например, `/users/:userId`).
            *   Получение значения параметра в компоненте с помощью хука `useParams`.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение навигации и динамических маршрутов.

## Неделя 5: Продвинутые Хуки и Оптимизация

*   **Урок 9: Хуки на Заказ (Custom Hooks) (2 часа)**
    *   **Теория (40 мин):** Идея создания переиспользуемой логики. Как создавать пользовательские хуки. Правила именования.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: Создание Простого Хука (40 мин)**
            *   Перенос логики из компонента в пользовательский хук (например, хук `useToggle` для переключения булева состояния).
            *   Использование созданного хука в нескольких компонентах.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение создания хуков.
        *   **Задача 2: Хук для Загрузки Данных (35 мин)**
            *   Рефакторинг логики получения данных с индикатором загрузки и ошибкой из Урока 5 в пользовательский хук `useFetch`.
            *   Использование `useFetch` в компоненте.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение рефакторинга и более сложных хуков.

*   **Урок 10: Оптимизация Производительности (Базово) (2 часа)**
    *   **Теория (40 мин):** Когда нужно оптимизировать. `React.memo` для мемоизации функциональных компонентов (поверхностное сравнение пропсов). `useMemo` для мемоизации вычислений. `useCallback` для мемоизации функций.
    *   **Практика (1 час 20 мин - Разбито на блоки):**
        *   **Задача 1: `React.memo` (30 мин)**
            *   Создание компонента, который медленно рендерится (например, выполняющего долгие вычисления).
            *   Обворачивание компонента в `React.memo` и демонстрация эффекта при изменении пропсов, которые не должны вызывать перерендер.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение `React.memo`.
        *   **Задача 2: `useMemo` и `useCallback` (45 мин)**
            *   Применение `useMemo` к результатам вычислений.
            *   Применение `useCallback` к колбэк-функциям, передаваемым в компоненты, обернутые в `React.memo`.
            *   Демонстрация сценариев, где эти хуки помогают.
        *   **Обсуждение и Вопросы (5 мин):** Обсуждение `useMemo` и `useCallback`.

**Важные Примечания для Интенсивного Курса:**

*   **Предварительная Подготовка:** Студентам, возможно, потребуется выполнять больше самостоятельной работы или ознакомиться с некоторыми материалами перед уроками.
*   **Готовый Стартовый Код:** Предоставление готового кода для каждой практической задачи сэкономит время и позволит сосредоточиться на новой концепции.
*   **Активное Участие:** Интенсивный курс требует высокой вовлеченности как преподавателя, так и студентов.
*   **Домашние Задания:** Небольшие домашние задания между уроками критически важны для закрепления материала в быстром темпе.
*   **Фокус на Ключевых Идеях:** Приходится жертвовать глубиной в некоторых темах ради охвата широкого спектра.

Этот план более плотный и требует более быстрого освоения материала, но сохраняет структуру с интегрированной практикой, разбитой на управляемые части.
